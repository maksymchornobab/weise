Це чудова мета. Щоб твій проект не залишився просто набором коду, а став повноцінним інструментом, який легко розгорнути на іншому ПК, ми використаємо **Rust** як основний двигун та **Cargo** (менеджер пакетів Rust) для збірки.

Ось твій детальний технічний маршрут у VS Code.

---

## Крок 1: Підготовка середовища (Dependencies)

Перш ніж писати код, потрібно підготувати "фундамент" на обох машинах.

### 1. Системні інструменти

* **Rust & Cargo:** Встанови через [rustup.rs](https://rustup.rs/). Це встановить компілятор і менеджер проектів.
* **C++ Build Tools:** (Тільки для Windows) Rust потребує лінкер Visual Studio.
* **VS Code Extensions:**
* `rust-analyzer` (обов'язково — це "мізки" для коду).
* `CodeLLDB` (для дебагу).



### 2. Створення проекту

Відкрий термінал у VS Code і створи проект:

```bash
cargo new invisible-p2p
cd invisible-p2p

```

### 3. Налаштування `Cargo.toml`

Додай ці залежності у файл `Cargo.toml`. Вони реалізують той самий L1 стелс-стек:

```toml
[dependencies]
libp2p = { version = "0.53", features = ["tcp", "quic", "noise", "yamux", "kad", "identify", "autonat", "hole-punching"] }
tokio = { version = "1.0", features = ["full"] } # Асинхронний двигун
futures = "0.3"
serde = { version = "1.0", features = ["derive"] } # Для серіалізації даних
ed25519-dalek = "2.0" # Криптографічні підписи
rand = "0.8" # Для генерації шуму (Padding)

```

## Крок 2: Розробка архітектури (L1 Logic)

Тобі потрібно реалізувати три головні модулі в `src/main.rs`:

1. **Transport:** Налаштування `libp2p` з шифруванням **Noise**.
2. **Network Behaviour:** Опис того, як вузли знаходять один одного (Kademlia DHT) і як вони обмінюються повідомленнями (Gossipsub або Floodsub).
3. **Stealth Layer:** Функція, яка бере рядок тексту, додає до нього випадкові байти до фіксованого розміру (наприклад, 1024 байти) перед відправкою.

---

## Крок 3: Компіляція та "Обгортка" (Packaging)

Щоб перенести програму на інший ПК без встановлення там Rust, ми зробимо **Static Binary** (один файл, де все всередині).

### 1. Збірка для своєї ОС

У терміналі VS Code:

```bash
cargo build --release

```

Твій готовий додаток з'явиться за шляхом: `target/release/invisible-p2p` (або `.exe` на Windows).

### 2. Крос-компіляція (якщо другий ПК на іншій ОС)

Якщо ти пишеш на Windows, а другий ПК — Mac або Linux, використай `cross`:

```bash
cargo install cross
cross build --target x86_64-unknown-linux-gnu --release

```

## Крок 4: Як зробити "додаток, який можна завантажити"

Щоб це виглядало як справжній софт, а не просто скрипт:

* **Портативність:** Тобі не потрібен інсталятор. Просто скопіюй файл із папки `release` на флешку або скинь через хмару.
* **Конфігурація:** Створи поруч із файлом `.env` або `config.toml`, де ти зможеш прописати `BOOTSTRAP_NODE` (IP-адресу першого комп'ютера), щоб вони "побачили" один одного в інтернеті.
* **GUI (Опціонально):** Якщо захочеш вікно замість консолі, пізніше додамо бібліотеку **Tauri** (вона дозволяє робити інтерфейс на HTML/JS, а бекенд на Rust — виходить дуже легко і швидко).

3. **Запусти через Wi-Fi:** Спробуй `cargo run` на основному ПК і скопіюй бінарний файл на другий ПК.

> **Важливо щодо L1:** Оскільки ми не використовуємо центральний сервер, першому ПК ("Вузол А") доведеться повідомити свою публічну адресу другому ("Вузол Б"). У реальному L1 це робиться через "Bootnodes" (статичні адреси). Для тесту ти можеш просто вписати IP свого роутера вручну.
